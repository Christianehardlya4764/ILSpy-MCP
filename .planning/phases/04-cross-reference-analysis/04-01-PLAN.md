**FindImplementorsAsync (XREF-02) -- type system traversal, NOT IL scanning (per D-10):**
- Walk `mainModule.TypeDefinitions.Where(t => t.ParentModule == mainModule)`
- For each type, check `type.DirectBaseTypes` for match against `typeName.FullName`

- Return `List<ImplementorResult>` with TypeFullName and Relationship

**FindUsagesAsync (XREF-01), FindDependenciesAsync (XREF-03), FindInstantiationsAsync (XREF-04) -- IL scanning:**
All three use the shared IL scanning loop. Create private helper methods (per D-11):

1. **Private method `ScanAllMethodBodies`** -- iterates every type definition, every method, reads IL:
   ```
   foreach typeDefHandle in metadataReader.TypeDefinitions
     foreach methodDefHandle in typeDef.GetMethods()
       skip if methodDef.RelativeVirtualAddress == 0 (abstract/extern)
       get MethodBodyBlock via metadataFile.GetMethodBody(rva)
       get BlobReader via methodBody.GetILReader()
       while ilReader.RemainingBytes > 0: decode opcode, process operand
   ```

2. **Private method `SkipOperand(ref BlobReader reader, ILOpCode opCode)`** -- CRITICAL for correct IL traversal. Must handle ALL operand types from `opCode.GetOperandType()`:
   - `InlineNone`: 0 bytes
   - `ShortInlineBrTarget` / `ShortInlineI` / `ShortInlineVar`: `reader.ReadByte()` (1 byte)
   - `InlineVar`: `reader.ReadInt16()` (2 bytes)
   - `InlineBrTarget` / `InlineField` / `InlineI` / `InlineMethod` / `InlineSig` / `InlineString` / `InlineTok` / `InlineType`: `reader.ReadInt32()` (4 bytes)
   - `InlineI8` / `InlineR`: `reader.ReadInt64()` (8 bytes)
   - `InlineSwitch`: `int count = reader.ReadInt32(); reader.Offset += count * 4;`
   - `ShortInlineR`: `reader.ReadSingle()` (4 bytes)
   Use the exact code from 04-RESEARCH.md SkipOperand helper.

3. **Private method `ResolveToken(MetadataReader reader, EntityHandle handle)`** returning `(string DeclaringType, string MemberName)?`:
   - Handle `HandleKind.MethodDefinition`: get declaring type, get method name
   - Handle `HandleKind.MemberReference`: get parent name, get member name
   - Handle `HandleKind.MethodSpecification`: recurse to `methodSpec.Method` (generic resolution per Pitfall 2)
   - Handle `HandleKind.FieldDefinition`: get declaring type, get field name
   - Return null for unrecognized handle kinds
   Use the exact code from 04-RESEARCH.md Token Resolution Helper, extended for FieldDefinition.

4. **Private method `GetFullTypeName(MetadataReader reader, TypeDefinition typeDef)`** returning string:
   - Combine namespace + "." + name
   - Handle empty namespace (just name)

5. **Private method `ResolveParentName(MetadataReader reader, EntityHandle parent)`** returning string:
   - Handle `TypeDefinitionHandle`: use GetFullTypeName
   - Handle `TypeReferenceHandle`: combine namespace + "." + name
   - Handle `TypeSpecificationHandle`: return generic type name (best effort)
   - Default: return parent.Kind.ToString()

6. **Private method `GetMethodSignature(MetadataReader reader, MethodDefinition methodDef)`** returning string:
   - Format: `ReturnType MethodName(ParamType1, ParamType2)`
   - Use `methodDef.DecodeSignature(...)` or build from parameter names -- keep it simple, human-readable

**FindUsagesAsync implementation (per D-04, D-06, D-07):**
- Validate member exists on type (find method/field/property with matching name). If not found, throw `MemberNotFoundException(memberName, typeName.FullName)`
- For properties, resolve to getter/setter method names (`get_PropertyName`, `set_PropertyName`) per Pitfall 3
- Scan all method bodies, looking for opcodes: `Call`, `Callvirt`, `Newobj`, `Ldfld`, `Stfld`, `Ldsfld`, `Stsfld`, `Ldftn`, `Ldvirtftn`
- For each matching token, resolve via `ResolveToken`, compare member name against target member name(s) and declaring type against target type
- Build `UsageResult` with containing type, containing method signature, IL offset, and opcode name (lowered, e.g. "callvirt")

**FindDependenciesAsync implementation (per D-06, D-07):**
- If `memberName` is provided, scan only that method's body. If null, scan all methods in the type.
- For each method body, collect ALL member reference opcodes (same list as usages)
- Build `DependencyResult` with target type, target member, IL offset, opcode

**FindInstantiationsAsync implementation (per D-04):**
- Scan all method bodies, looking specifically for `ILOpCode.Newobj`
- Resolve the constructor's declaring type via `ResolveToken`
- Match declaring type against `typeName.FullName`
- Build `InstantiationResult` with containing type, containing method signature, IL offset

**Error handling (follow ILSpyDisassemblyService pattern):**
- Wrap each method in `Task.Run(() => { ... }, cancellationToken)`
- Catch `TypeNotFoundException` and `MemberNotFoundException` -- re-throw
- Catch `FileNotFoundException` -- throw `AssemblyLoadException`
- Catch generic `Exception` -- log and throw `AssemblyLoadException`
