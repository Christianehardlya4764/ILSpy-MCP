```csharp
public sealed record AssemblyPath
{
    public string Value { get; }
    private AssemblyPath(string value) => Value = value;
    public static AssemblyPath Create(string path)
    {
        if (string.IsNullOrWhiteSpace(path))
            throw new ArgumentException("Assembly path cannot be null or empty.", nameof(path));
        if (!File.Exists(path))
            throw new FileNotFoundException($"Assembly file not found: {path}", path);
        if (!Path.GetExtension(path).Equals(".dll", StringComparison.OrdinalIgnoreCase) &&
            !Path.GetExtension(path).Equals(".exe", StringComparison.OrdinalIgnoreCase))
            throw new ArgumentException($"Invalid assembly file extension: {path}", nameof(path));
        return new AssemblyPath(Path.GetFullPath(path));
    }
    public string FileName => Path.GetFileName(Value);
}
```

    - Class `ILSpyCrossAssemblyService : ICrossAssemblyService` in namespace `ILSpy.Mcp.Infrastructure.Decompiler`
    - Constructor takes `ILogger<ILSpyCrossAssemblyService>`
    - Private static helper `EnumerateAssemblyFiles(string root, int maxDepth)` returns `IEnumerable<string>`:
      - Enumerate files in `root`, yield those ending in `.dll` or `.exe` (case-insensitive) per D-11
      - If `maxDepth > 0`, recurse into subdirectories with `maxDepth - 1`
      - This avoids `SearchOption.AllDirectories` which has no depth limit
    - **LoadAssemblyDirectoryAsync**: Enumerate files via `EnumerateAssemblyFiles(directoryPath.Value, maxDepth)`. For each file, try `new PEFile(file)` (lightweight, no full type system). Read `MetadataReader`, get `AssemblyDefinition` for name/version. Count `TypeDefinitions`. Add to `loaded` list. Catch `PEFileNotSupportedException` (or other exceptions from non-managed files) -> add to `skipped` with reason. Return `DirectoryLoadResult` with both lists. Dispose each `PEFile` after reading.
    - **ResolveTypeAsync**: Enumerate files via `EnumerateAssemblyFiles`. For each file, try creating `CSharpDecompiler(filePath, new DecompilerSettings())`. Iterate `decompiler.TypeSystem.MainModule.TypeDefinitions`. Match per D-14: `type.Name.Equals(typeName, OrdinalIgnoreCase)` OR `type.FullName.Contains(typeName, OrdinalIgnoreCase)`. Collect all matches as `TypeResolutionResult` with file path, assembly name, full/short type name. Catch exceptions for unloadable files and skip. Return all matches (per D-13, return ALL matching assemblies).

    - Then list loaded: `  {AssemblyName} v{Version} ({TypeCount} types) - {FilePath}`
    - If skipped count > 0: section "Skipped:" with `  {FilePath}: {Reason}`
