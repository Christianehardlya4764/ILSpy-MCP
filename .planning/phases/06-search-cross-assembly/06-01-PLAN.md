    **Infrastructure/Decompiler/ILSpySearchService.cs** (per D-05, D-06, D-07):
    - Class `ILSpySearchService : ISearchService` in namespace `ILSpy.Mcp.Infrastructure.Decompiler`
    - Constructor takes `ILogger<ILSpySearchService>`
    - Duplicate the IL scanning helpers from ILSpyCrossReferenceService: `ReadILOpCode`, `SkipOperand`, `GetOperandSize` as private static methods (research recommends duplication over extraction)
    - Also duplicate `FormatMethodSignature` and `GetTypeFullName` helper methods
    - **SearchStringsAsync**: Create `PEFile` from assemblyPath.Value, get `MetadataReader`, iterate all `TypeDefinitionHandle`, for each type iterate all `MethodDefinitionHandle`, for each method get `MethodBodyBlock` via `metadataFile.GetMethodBody(rva)`, iterate IL with `ReadILOpCode`/`SkipOperand`, when `opCode == ILOpCode.Ldstr` read 4-byte token with `ilReader.ReadInt32()`, extract UserString via `MetadataTokens.UserStringHandle(token & 0x00FFFFFF)` then `reader.GetUserString(handle)`, match against compiled `Regex(regexPattern, RegexOptions.Compiled)`, collect all matches, then apply offset/limit pagination, return `SearchResults<StringSearchResult>` with `TotalCount` = total matches before pagination
    - **SearchConstantsAsync**: Same method body iteration pattern, but check for `Ldc_i4_m1` (-1), `Ldc_i4_0` through `Ldc_i4_8` (0-8), `Ldc_i4_s` (ReadSByte, NOT ReadByte), `Ldc_i4` (ReadInt32), `Ldc_i8` (ReadInt64). Compare extracted value against target `long value`. Do NOT include `Ldc_r4`/`Ldc_r8` (float/double -- exact match unreliable). Set `ConstantType` to "Int32" for i4 variants, "Int64" for i8. Apply offset/limit pagination.
    - Skip methods with `RelativeVirtualAddress == 0` (abstract/extern methods have no body)
    - Wrap `PEFile` in using statement for proper disposal
    - Use `CSharpDecompiler` with `new DecompilerSettings()` for type system access to get type/method names (same pattern as ILSpyCrossReferenceService)
