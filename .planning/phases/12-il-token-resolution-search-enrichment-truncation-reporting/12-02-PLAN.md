   Update `ScanILForStrings` to capture surrounding IL instructions. The approach (per Claude's discretion): capture during the existing scan pass for efficiency rather than re-scanning. Strategy:
   - Before the main scan loop, do a pre-pass to collect all instruction boundaries and their formatted representations. Create a list of `(offset, formattedInstruction)` tuples by iterating the IL bytes once, formatting each instruction as `IL_{offset:X4}: {opcode} {operand}`.
   - For operands that reference metadata tokens (call, callvirt, newobj, ldfld, etc.), resolve the token to a readable name using the MetadataReader. Use `MetadataTokens.EntityHandle(token)` to get the handle, then resolve to a member name via the decompiler's type system or raw metadata reader.
   - Then do the ldstr-matching scan using the pre-built instruction list. When a match is found, take up to 3 instructions before and 3 after from the list.
   - Alternative simpler approach: after finding an ldstr match, record the surrounding instruction offsets. Then use `ReflectionDisassembler` with a `PlainTextOutput(StringWriter)` to disassemble the entire method body, split the output into lines, find the matching IL_xxxx offset line, and extract a window. This is less efficient but much simpler and leverages ReflectionDisassembler's existing token resolution.

   **Recommended approach** (simpler, leverages existing infrastructure): After scanning finds matches, for each matched method, disassemble the method body once using `ReflectionDisassembler`, split into lines, then for each match in that method extract the surrounding IL window. Cache the disassembly per method to avoid redundant work.

   Implementation:
   - Add a new private method `CaptureMethodILLines(MetadataFile metadataFile, MethodDefinitionHandle methodHandle, CancellationToken ct)` that returns `List<(int offset, string line)>`:
     ```csharp
     private static List<(int offset, string line)> CaptureMethodILLines(
         MetadataFile metadataFile, MethodDefinitionHandle methodHandle, CancellationToken ct)
     {
         using var writer = new StringWriter();
         var output = new PlainTextOutput(writer);
         var disassembler = new ReflectionDisassembler(output, ct) { DetectControlStructure = false };
         disassembler.DisassembleMethod(metadataFile, methodHandle);
         var lines = writer.ToString().Split('\n', StringSplitOptions.RemoveEmptyEntries);
         var result = new List<(int, string)>();
         foreach (var line in lines)
         {
             var trimmed = line.TrimStart();
             if (trimmed.StartsWith("IL_") && trimmed.Length >= 7)
             {
                 if (int.TryParse(trimmed.Substring(3, 4), System.Globalization.NumberStyles.HexNumber, null, out int offset))
                 {
                     result.Add((offset, trimmed.TrimEnd()));
                 }
             }
         }
         return result;
     }
     ```
   - After `ScanILForStrings` completes, for each unique method in the results, call `CaptureMethodILLines` once and then for each match in that method, extract a window of 3 before + match + 3 after from the IL lines list.
   - Update the match creation to populate `SurroundingInstructions` and `MatchInstructionIndex`.
   - Add necessary `using` directives: `ICSharpCode.Decompiler.Disassembler`, `ICSharpCode.Decompiler.Output`.
